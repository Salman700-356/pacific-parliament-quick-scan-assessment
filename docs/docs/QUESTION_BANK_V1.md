# PPQSA Question Bank v1

Scoring:
0 = Not in place  
1 = Ad hoc / inconsistent  
2 = Mostly in place / consistent  
3 = Managed / measurable  
N/A = Not applicable (excluded from averages)

## GOV — Governance & Ownership
GOV-01 Is there a named person/team accountable for cyber security?
GOV-02 Does cyber risk get reported to leadership at least quarterly?
GOV-03 Are basic security policies in place (acceptable use, passwords, remote access, data handling)?
GOV-04 Is there a known set of top cyber risks (risk register or risk list)?

## AST — Asset Visibility
AST-01 Do you maintain a list of ICT assets (endpoints/servers/network equipment)?
AST-02 Do you know what is public-facing/internet exposed (websites, VPN, email, DNS, hosting)?
AST-03 Is there a defined way to track patching of servers/endpoints (even basic)?
AST-04 Have you identified which systems are critical to Parliament operations?

## IAM — Identity & Access Control
IAM-01 Is MFA enabled for email and remote access for all staff?
IAM-02 Are admin accounts separated from normal user accounts?
IAM-03 Are access changes and leavers handled promptly and consistently?
IAM-04 Are privileged accounts reviewed regularly (at least every 6–12 months)?

## END — Endpoint Security
END-01 Are workstations/laptops centrally managed (Intune/MDM/GPO/standard build)?
END-02 Is endpoint protection deployed and monitored (Defender, CrowdStrike, Sophos etc.)?
END-03 Are all portable devices encrypted (BitLocker/FileVault)?
END-04 Are local admin rights restricted and granted only where necessary?

## PER — Email & Perimeter Controls
PER-01 Is phishing/spam filtering in place and actively managed?
PER-02 Are SPF + DKIM + DMARC implemented for official Parliament domains?
PER-03 Is remote access protected (VPN or ZTNA + MFA, strong controls)?
PER-04 Is there a managed firewall with change control and basic rule hygiene?

## BAK — Backups & Resilience
BAK-01 Are backups in place for critical systems and important data?
BAK-02 Are restores tested at least annually (or more often)?
BAK-03 Are backups protected from ransomware (offline/immutable/separate credentials)?
BAK-04 Is there a basic continuity plan or agreed approach for what matters most during outages?

## LOG — Logging & Monitoring
LOG-01 Is there central visibility of security events (SIEM, Defender portal, syslog, alerting)?
LOG-02 Is there a defined process/person responsible for responding to alerts?
LOG-03 Are key audit logs enabled for identity/email/admin activity?

## IR — Incident Response & Awareness
IR-01 Is there a basic documented incident response process (even a 1–2 pager)?
IR-02 Have you run an incident tabletop exercise in the last 12–18 months?
IR-03 Do staff receive security awareness guidance/training at least annually?
